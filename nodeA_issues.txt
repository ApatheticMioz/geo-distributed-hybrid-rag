Node A alignment issues vs Node C + report

1) Uses HTTP JSON instead of gRPC bidi stream.
   - Node A exposes /generate (FastAPI) and expects JSON.
   - Node C uses gRPC GenerateStream (bidirectional streaming).

2) No gRPC server for GenerateStream or SparseContextRequest.
   - Missing support for node_b_dispatch_failed flag.

3) No receiver for Node B dense results over LAN.
   - Report requires Node B to push dense results directly to Node A.

4) T_THRESHOLD_MS defined but unused.
   - No timeout or fallback logic for dense branch delays.

5) RRF input mismatch.
   - Node A expects list of doc_id strings from JSON.
   - Node C sends RetrievedDocument (doc_id, text, score, rank).

6) No overlap of sparse and dense branches.
   - Current flow waits for a single request body; no parallel branch handling.

7) Requirements include httpx.
   - Inter-node communication should be gRPC only.

8) Port/protocol mismatch with Node C.
   - Node A serves HTTP on 8001, but Node C expects gRPC on 50052.

9) No query_id handling or correlation for multi-branch requests.
   - Node C generates query_id and expects it to flow through Node A.
